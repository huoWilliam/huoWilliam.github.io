发表于 2020-07-28 更新于 2020-07-31 分类于 读书总结
讲解对文件与目录的操作，包括在不同的目录间变换、 创建与删除目录、创建与删除文件，还有寻找文件、查阅文件内容等等。

目录与路径

下达指令时，该指令通过何种方式取得？这有PATH这个系统变量有关。

绝对路径与相对路径

  1. 绝对路径：路径的写法“一定由根目录 / 写起”，例如： /usr/share/doc 这个目录。

  2. 相对路径：路径的写法“不是由 / 写起”，例如由 /usr/share/doc 要到 /usr/share/man 下面时，可以写成： “cd ../man”这就是相对路径的写法啦！相对路径意指“相对于目前工作目录的路径！”

目录的相关操作

有一些特殊的目录需要记忆下来，如下所示。

.：表示当前目录

..：表示上一层目录

-：表示前一个工作目录

~：表示当前用户的主文件夹

~account：表示account这个用户的主文件夹

修改文件时间或创建新文件——touch

Linux中的文件有三个主要的时间参数，分别含义如下：

  1. mtime：当该文件的“内容数据”变更时，就会更新这个时间！内容数据指的是文件的内容，而不是文件的属性或权限喔！默认情况下，ls显示的就是文件的mtime；
  2. ctime：当该文件的“状态 （status）”改变时，就会更新这个时间，举例来说，像是权限与属性被更改了，都会更新这个时间啊。
  3. atime：当“该文件的内容被取用”时，就会更新这个读取时间 （access）。举例来说，我们使用 cat 去读取 /etc/man_db.conf ， 就会更新该文件的 atime 了。

文件特殊权限

  1. SUID

    当 s 这个标志出现在文件拥有者的 x 权限位置上时，就被称为 Set UID，简称为 SUID 的特殊权限。SUID具有这样的功能与限制：

    • SUID 权限仅对二进制程序（binary program）有效，对目录无效；
    • 执行者对于该程序需要具有 x 的可执行权限；
    • 本权限仅在执行该程序的过程中有效 （run-time）；
    • 执行者将具有该程序拥有者 （owner） 的权限。

  2. SGID

    当s这个标志出现在群组的x权限位置上时，就称为Set GID，简称SGID的特殊权限。不同于SUID，SGID可以对文件或目录进行设置。对于文件来说，SGID具有如下的功能：

    • SGID 对二进制程序有用；
    • 程序执行者对于该程序来说，需具备 x 的权限；
    • 执行者在执行的过程中将会获得该程序群组的支持！

    对于目录来说，SGID具有如下的功能：

    • 使用者若对于此目录具有 r 与 x 的权限时，该使用者能够进入此目录；
    • 使用者在此目录下的有效群组（effective group）将会变成该目录的群组；
    • 用途：若使用者在此目录下具有 w 的权限（可以新建文件），则使用者所创建的新文件，该新文件的群组与此目录的群组相同。

  3. SBIT

    SBIT目前仅针对目录有效。它对于目录的作用是：

    • 当使用者对于此目录具有 w, x 权限，亦即具有写入的权限时；
    • 当使用者在该目录下创建文件或目录时，仅有自己与 root 才有权力删除该文件

  4. SUID/SGID/SBIT权限设置

    在三个权限数字前再加上一个数字，利用该数字来设置这些特殊权限。

    记住，4为SUID，2为SGID，3为SBIT。

    假设要将一个文件权限改为“-rwsr-xr-x”时，由于 s 在使用者权限中，所以是 SUID ，因此， 在原先的 755 之前还要加上 4 ，也就是：“ chmod 4755 filename ”来设置！
